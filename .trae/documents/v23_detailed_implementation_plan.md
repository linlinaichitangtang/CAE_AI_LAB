# CAELab V2.3 详细实施计划

## 1. 项目背景
V2.3 是 Phase 3 的求解器补全版本，目标是在 V2.2 已完成的 VS 风格安装框架基础上，填入真实求解器逻辑，并解决 V2.2 遗留的 KI-001~004 问题。

## 2. 当前状态
| 求解器 | SolverDef | 检测命令 | 实际功能 |
|--------|-----------|---------|--------|
| LAMMPS | ✅ | `lmp_serial --version` | ✅ V2.2 已完成 |
| Quantum ESPRESSO | ✅ | `pw.x --version` | ⚠️ 仅框架，任务提交/结果解析未实现 |
| FiPy | ✅ | `python3 -c "import fipy..."` | ⚠️ 仅框架，CH方程求解未实现 |
| OpenFOAM | ✅（本次新增） | `foamExec -help` | ⚠️ 仅框架，任务提交/结果解析未实现 |
| CalculiX | ✅ | `ccx -h` | ⚠️ 仅框架；且 KI-001（2.17版本bug）未解决 |

## 5. 约束条件

1. **不破坏现有框架**：所有新增求解器必须走 V2.2 已建的 `solver_manager.rs` 安装框架
2. **npm run build 通过**：前端每次变更后运行 `npm run build`，零 error
3. **cargo build --release 通过**：Rust 后端零 error
4. **任务提交不走 mock**：所有 `invoke()` 命令必须真实调用求解器进程，解析真实输出文件

## 8. 关键技术点

1. **CalculiX 2.20+ 编译**：解决 KI-001 bug，需要正确处理 SPOOLES 和 ARPACK 依赖
2. **QE 输入文件生成**：确保 INCAR/KPOINTS 参数正确转换为 pw.in 格式
3. **任务提交与轮询**：实现后台执行和实时状态更新
4. **结果解析**：准确提取求解器输出的关键数据
5. **数据桥接**：实现真实的粗粒化、单位换算和坐标映射
6. **标准算例验证**：确保求解器集成的准确性和可靠性

## 9. 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| CalculiX 依赖编译失败 | 影响 FE 求解器集成 | 提前准备依赖包，确保编译环境正确 |
| QE 执行性能问题 | 影响 DFT 计算效率 | 优化任务提交参数，考虑并行计算 |
| 数据桥接精度问题 | 影响多尺度模拟准确性 | 详细测试不同插值方法的精度 |
| 安装环境差异 | 影响跨平台兼容性 | 完善安装脚本，支持不同包管理器 |
