## 2. 总体原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | 增量发布 | 每 0.1 一个版本，每版均可演示、可运行 |
| 2 | 架构稳定 | 基础架构不变，渐进增强，不做破坏性重构 |
| 3 | 对齐 PRD | 与 PRD Phase 划分保持同步 |
| 4 | 移动优先 | **iPadOS / 安卓平板 / 鸿蒙 NEXT 为近期核心目标** |
| 5 | 质量优先 | 先稳定后功能，先可运行后丰富 |
| 6 | 用户优先 | **V0.10 起核心目标切换为"让用户用得更舒服"——注册/登录/会员/云同步** |

## 5. 移动端技术路线

### 5.1 平台覆盖策略

| 平台 | 目标设备 | 技术方案 | 复用率 | 构建命令 |
|------|---------|---------|-------|---------|
| **iPadOS** | iPad Pro / iPad Air / iPad mini | Tauri 2 iOS 构建（WKWebView） | 95% | `tauri ios build` |
| **Android 平板** | 小米平板 / 荣耀平板 / 华为平板（旧款） | Tauri 2 Android 构建（WebView） | 95% | `tauri android build` |
| **鸿蒙 NEXT** | 华为 MatePad 系列（纯血鸿蒙） | ArkTS 壳 + Web 组件加载 H5 | 85% | ArkTS 构建 |

### 5.2 关键技术决策

| # | 决策 | 理由 |
|---|------|------|
| 1 | 共享前端代码（~95%） | 降低维护成本，三平台同步迭代 |
| 2 | Tauri 2 统一构建 | 官方支持好，Rust 生态成熟 |
| 3 | 鸿蒙独立构建 | 纯血鸿蒙不兼容 Android，需 ArkTS 轻量容器 |
| 4 | 触控优先设计 | 移动端重新设计交互，44pt 最小点击区域 |
| 5 | 功能分层 | 移动端聚焦"查看 + 笔记 + 轻量仿真"，复杂操作在桌面端 |

### 5.3 开发环境要求

| 平台 | 操作系统 | 开发工具 | 运行时 | 构建目标 |
|------|---------|---------|-------|---------|
| iPadOS | macOS 13+ | Xcode 15+ | Rust 1.75+, Node 18+ | .ipa |
| Android | macOS / Linux / Windows | Android Studio + NDK r27b | Rust 1.75+, Node 18+ | .apk / .aab |
| 鸿蒙 NEXT | macOS / Windows / Linux | DevEco Studio 5+ | ArkTS 5+ | .hap |

users

├── avatar_url      VARCHAR(500)

├── created_at      TIMESTAMP
└── updated_at     TIMESTAMP

devices
├── id          UUID (PK)
├── user_id     UUID (FK → users.id)
├── name        VARCHAR(100)  -- "MacBook Pro 2023"
├── ip          VARCHAR(45)
├── user_agent  VARCHAR(500)
├── last_seen   TIMESTAMP
└── created_at  TIMESTAMP

audit_log
├── id         UUID (PK)
├── user_id    UUID (FK → users.id)
├── action     VARCHAR(100)  -- login/logout/password_change/subscription_change

├── device_id  UUID
└── created_at TIMESTAMP

### 1.8.1 核心理念

> 1. DFT 计算 Mg-Al 二元体系的基态能量和弹性常数
> 2. MD 用 DFT 训练的势函数模拟原子扩散和位错运动
> 3. 相场用 MD 的扩散系数模拟晶界析出相组织演变
> 4. FE 用相场的组织场做等效材料输入，模拟零件的蠕变变形
> **全程无需用户手动转换格式，全部自动完成。**
