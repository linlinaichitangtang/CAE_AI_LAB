# CAELab 版本路线图

## 总体原则
- 每 0.1 一个版本，稳扎稳打
- 每版均可演示、可运行
- 基础架构不变，渐进增强
- 与 PRD Phase 划分保持对齐
- **移动端（iPadOS / 安卓平板 / 鸿蒙 NEXT）为近期核心目标**

---

## 版本与 PRD Phase 对应关系

| 版本范围 | PRD Phase | 主题 |
|---------|-----------|------|
| V0.1 ~ V0.3 | Phase 1 前期 | 桌面端质量打磨 + 联动完善 |
| V0.4 ~ V0.5 | Phase 1 后期 | 桌面端工程可用 + V1.0 发布 |
| V0.6 ~ V0.8 | Phase 2 前期 | **移动端适配（iPadOS / 安卓 / 鸿蒙 NEXT）** |
| V0.9 ~ V1.2 | Phase 2 后期 | 能力增强（CFD / 协作 / 优化） |
| V1.5+ | Phase 3 | 平台扩展（详见远景规划） |

## 📱 移动端技术路线

> 移动端是近期核心目标，需要覆盖三大平台，技术路线如下：

### 平台覆盖与方案

| 平台 | 目标设备 | 技术方案 | 说明 |
|------|---------|---------|------|
| **iPadOS** | iPad Pro / iPad Air / iPad mini | Tauri 2 iOS 构建 | Tauri 2 已正式支持 iOS，基于 WKWebView |
| **Android 平板** | 小米平板 / 荣耀平板 / 华为平板（旧款）等 | Tauri 2 Android 构建 | Tauri 2 已正式支持 Android，基于 WebView |
| **鸿蒙 NEXT** | 华为 MatePad 系列（纯血鸿蒙） | WebView 容器 + ArkTS 壳应用 | 鸿蒙 NEXT 不兼容 Android，需用 ArkTS 开发轻量容器，加载前端 H5 资源 |

### 关键技术决策

1. **共享前端代码**：三个平台共用 Vue 3 前端代码（约 95% 复用），仅平台原生层不同
2. **Tauri 2 统一构建**：iPadOS 和 Android 平板通过 Tauri 2 的 `cargo-mobile2` 工具链构建
3. **鸿蒙 NEXT 独立构建**：将 Vite 构建产物（H5）打包为鸿蒙应用，通过 ArkTS 的 `<Web>` 组件加载
4. **触控优先**：移动端 UI 需要重新设计，手势操作、触控笔支持、响应式布局
5. **功能裁剪**：移动端初期聚焦"查看 + 笔记 + 轻量仿真"，复杂建模和大规模求解仍在桌面端

### 各平台开发环境要求

| 平台 | 开发环境 | 构建工具 |
|------|---------|---------|
| iPadOS | macOS + Xcode 15+ | `tauri ios build` |
| Android | Android Studio + NDK | `tauri android build` |
| 鸿蒙 NEXT | DevEco Studio 5+ | ArkTS 项目构建 |
