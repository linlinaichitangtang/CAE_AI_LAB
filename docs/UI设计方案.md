## 一、整体UI风格定位

### 1.1 设计哲学

**"静谧深空 · 精准工坊"**

> CAELab 是一款面向工程师和科研人员的专业工具。
> 它应该像一艘宇宙飞船的控制台——
> 深邃、沉静、功能清晰；
> 又像一间光线充足的精密实验室——
> 高效、准确、专注。

### 1.2 视觉关键词

| 关键词 | 含义 | 视觉表现 |
|--------|------|----------|
| **沉稳** | 专业可信，不花哨 | 深色基底、低饱和度点缀 |
| **精密** | 工具感、工程师气质 | 细线条、精密网格、精准对齐 |
| **通透** | 减少认知负担，信息分层清晰 | 微妙透明层次、阴影暗示 |
| **沉浸** | 让用户专注于创作内容 | 最小化UI噪音、上下文工具栏 |
| **科技感** | 前沿但克制 | 细粒度光效、精密感图标 |

### 1.3 设计语言

#### 色彩体系

**暗色主题（Dark Mode）- 默认**

```

  --primary:       #4F8CFF    // 探索蓝（主操作）
  --primary-glow:  rgba(79,140,255,0.15)  // 光晕
  
辅助色:
  --accent-cyan:   #22D3EE    // 科技青（3D/仿真）
  --accent-green:  #34D399    // 成功/运行态
  --accent-amber:  #FBBF24    // 警告/加载

```

#### 动效原则

```
持续时间:
  --duration-fast:   150ms   // 微交互（hover）
  --duration-normal: 250ms   // 标准过渡
  --duration-slow:   400ms   // 大幅动效（页面切换）

缓动曲线:
  --ease-out:    cubic-bezier(0.16, 1, 0.3, 1)
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1)  // 弹性效果
```

**动效哲学**: 快速、精准、不抢戏。动效是用来引导注意力，而非装饰。

## 二、整体应用结构

### 2.1 多窗格架构

CAELab 核心采用 **多窗格 (Multi-Pane)** 可拆分架构，适应不同创作场景：

```
┌─────────────────────────────────────────────────────────────────┐
│  顶部栏 (Title Bar) - 应用级操作                               │
├────────┬────────────────────────────────────────────┬───────────┤
│        │                                            │           │
│  左侧  │            主工作区                        │   右侧    │
│  导航  │        (可配置多窗格)                      │   属性    │
│  面板  │                                            │   面板    │
│        │                                            │           │
│  72px  │           flex-grow                       │   280px   │
│        │                                            │  (可折叠) │
├────────┴────────────────────────────────────────────┴───────────┤
│  底部状态栏 - 上下文信息 / 工具提示                             │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 窗格模式（Pane Modes）

```
1. 专注模式 (Focus Mode)
   - 单窗格，全屏
   - 适合笔记写作、代码编写

2. 并排模式 (Side-by-Side)
   - 主工作区 + 属性面板
   - 适合3D建模、仿真分析

3. 三分模式 (Tri-Panel)
   - 左侧导航 + 主工作区 + 右侧属性
   - 适合复杂的多工具协同

4. 四分模式 (Quad)
   - 四个等分窗格
   - 适合代码+预览、建模+仿真对照
```

#### 工具栏设计

**插入工具条** (Floating Toolbar):

```
┌────────────────────────────────────────────────────────────────┐
│ ➕ │ H1 H2 H3 │ **B** *I* ~~S~~ │ 📐 │ ∫∑ │ ``` │ 🎯 │ 📎 │
│   │  标题    │  粗体斜体删除  │ 表格 │ 公式 │ 代码块 │ 图形 │ 附件 │
└────────────────────────────────────────────────────────────────┘
```

**手写工具条** (Drawing Toolbar):

```
┌────────────────────────────────────────────────────────────────┐
│ ✏️ │ ● │ ◯ │ / │ ▢ │ ⬡ │ ↑ │ ─ │ 颜色▼ │ 粗细▼ │ ↩️ │ ➡️ │
│   │点│圆│直线│矩形│六边│箭头│曲线│线宽  │       │撤销│重做│
└────────────────────────────────────────────────────────────────┘
```

#### 交互细节

1. **自由手写区域**: 任何空白处按住触控笔即为手写模式
2. **压感支持**: 笔触粗细随压力变化（2048级压感）
3. **墨迹转文字**: 手写完成后长安0.5s弹出转换选项
4. **公式识别**: 手绘 ∫、∑、π 等符号自动识别为LaTeX
5. **图形标注**: 画图形后自动识别（矩形、圆形、三角形、箭头）

### 5.2 笔记创建流程

```
1. 点击 "+" 或快捷键 Cmd+N
   ↓
2. 选择文档类型：
   ├── 📝 空白笔记
   ├── 📄 Markdown
   └── 📋 模板库
   ↓
3. 进入编辑器
   ├── 自动创建空白页
   └── 光标定位于标题
   ↓
4. 开始创作
   ├── 手写 → 手写工具激活
   ├── 打字 → 键盘输入
   └── 插入 → 工具栏选择
   ↓
5. 自动保存（本地）+ 云同步（已登录）
   ↓
6. 完成编辑 → 关闭或返回首页
```

### 5.3 3D建模流程

```
1. 从导航进入「建模」模块
   ↓
2. 新建项目 / 打开已有
   ↓
3. 主界面：3D视口 + 左侧场景树 + 右侧属性面板
   ↓
4. 添加几何体
   ├── 方法A：工具栏 → 选择基本体（立方体/球/圆柱/锥）
   ├── 方法B：快捷键 → 数字键1-6选择
   └── 方法C：导入 → 加载STL/OBJ/STEP文件
   ↓
5. 编辑几何
   ├── 选择 → 点击对象
   ├── 变换 → 移动/旋转/缩放
   ├── 修改 → 拉伸/倒角/布尔
   └── 材质 → 属性面板选择
   ↓
6. 预览/渲染
   ├── 实时渲染模式
   └── 光线追踪（高清）
   ↓
7. 导出
   ├── 格式：STL, OBJ, STEP, FBX
   └── 用途：3D打印、游戏、CAE仿真
```
