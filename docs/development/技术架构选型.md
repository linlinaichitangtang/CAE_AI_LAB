## 1. 技术选型总览

### 1.1 跨平台框架选型

| 选项 | 框架 | 优势 | 劣势 | 推荐度 |
|------|------|------|------|--------|
| **A** | **Tauri 2.0** | 原生性能、Rust后端、安全、沙盒化、Web技术前端 | 生态较新、学习曲线 | ⭐⭐⭐⭐⭐ |
| B | Electron | 生态成熟、社区大、插件丰富 | 体积大、资源占用高 | ⭐⭐⭐ |
| C | Flutter | 原生渲染、跨平台最强 | 主要移动端，桌面端不成熟 | ⭐⭐⭐ |
| D | React Native | 前端熟悉的React | 桌面端支持有限 | ⭐⭐ |

**推荐方案：Tauri 2.0**

理由：
1. **性能优秀**：Rust后端，媲美原生应用
2. **体积小巧**：比Electron小3-5倍
3. **安全**：沙盒隔离，本地优先
4. **技术栈统一**：前端使用Web技术（Vue/React），团队学习成本低
5. **本地存储**：原生支持SQLite，适合CAELab本地优先需求

### 1.2 前端技术栈

| 层次 | 技术 | 选型理由 |
|------|------|----------|
| **UI框架** | Vue 3 + TypeScript | 组合式API、类型安全、生态成熟 |
| **状态管理** | Pinia | Vue官方推荐、轻量、支持持久化 |
| **3D渲染** | Three.js + @react-three/fiber | WebGL事实标准、丰富的生态系统 |
| **富文本编辑器** | TipTap (基于ProseMirror) | 高度可扩展、支持协同编辑 |
| **代码编辑器** | Monaco Editor | VSCode同款、强大的语法高亮和补全 |
| **手写识别** | 第三方库 + 自研 | WebCanvas API 实现手写输入 |
| **LaTeX渲染** | KaTeX | 快速、轻量、MathJax替代品 |
| **样式方案** | CSS Variables + Tailwind CSS | 主题切换、响应式、原子化 |

### 1.3 后端/计算层

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| **本地计算** | Rust原生 | Tauri后端，负责文件IO、本地计算任务 |
| **CAE求解器** | 外部集成 | Phase 1阶段使用外部开源求解器（如CalculiX、Elmer） |
| **脚本执行** | Python (embedded) | 通过Rust调用Python解释器执行工程脚本 |
| **Web服务** | Actix-web (Rust) | 轻量、高性能、必要时的本地Web服务 |
| **数据库** | SQLite + sqlx | 本地优先、跨平台、事务支持 |

**联动实现要点：**
1. **统一数据模型**：所有模块共享项目上下文

3. **变更感知**：模块间通过事件系统感知数据变化
4. **状态恢复**：跳转时保存上下文，返回时恢复

## 3. 关键技术决策

### 3.1 本地优先 + 云同步策略

**本地优先原则：**
- 所有数据首先写入本地SQLite
- 用户操作无需联网即可完成
- 联网后自动同步到云端

**同步策略：**
```
用户操作 → 本地写入 → 标记待同步 → 后台同步 → 确认同步成功
                ↓
          冲突检测 → 提示用户选择 → 最终合并
```

### 3.2 笔记内嵌对象机制

**嵌入数据格式（JSON）：**
```json
{
  "type": "embedded",
  "subtype": "3d-model",
  "refId": "model-uuid-001",
  "preview": "base64-encoded-preview",
  "position": { "x": 100, "y": 200 },
  "size": { "width": 300, "height": 200 }
}
```

**跳转机制：**
1. 用户点击嵌入对象
2. 解析refId和subtype
3. 加载对应模块
4. 恢复上下文（选中对象、位置等）

### 3.3 CAE求解器集成方案

**Phase 1策略：外部求解器**
- 使用开源求解器（如CalculiX）
- 通过命令行调用
- 结果解析与可视化在本地完成

**长期规划：自研/集成商业求解器**
- 根据商业模式决定精度定位
- 教学演示级：轻量化求解器
- 工程参考级：集成专业求解器

### 3.4 跨平台适配策略

| 平台 | 适配策略 |
|------|----------|
| iPad | 触控优先、手写支持、外接键盘支持 |
| macOS | 键鼠优化、Touch Bar支持 |
| Windows | 键鼠优化、主题跟随系统 |
| Linux | GTK/Qt混合，确保核心功能 |
